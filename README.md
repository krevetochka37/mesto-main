## Путешествия по России

Одностраничный адаптивный сайт о путешествиях по России. Проект выполнен на чистом HTML и CSS в методологии БЭМ, без JavaScript и без сборщиков — это набор статических файлов, готовых к публикации на любом хостинге статики (GitHub Pages, Netlify, Vercel и т.п.).

### Технологии и особенности

- **HTML5**: семантическая разметка.
- **CSS3**: flexbox, grid, позиционирование, адаптивная вёрстка.
- **Методология БЭМ**: файловая структура по блокам в папке `blocks`.
- **Normalize.css**: базовое выравнивание стилей в разных браузерах.
- **Кастомные шрифты**: подключены из папки `fonts`.

### Структура проекта

- `index.html` — основная HTML‑страница.
- `pages/index.css` — точка входа для стилей, в нём подключаются все блоки.
- `blocks/` — стили для блоков по БЭМ (`header`, `lead`, `intro`, `photo-grid`, `places`, `cover`, `footer` и др.).
- `fonts/` — файлы шрифтов (`woff`, `woff2`).
- `vendor/normalize.css` — нормализация стилей.

### Как запустить локально

1. **Склонировать или скачать** проект.
   - Если проект на GitHub:
     ```bash
     git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
     cd russian-travel-master
     ```
   - Либо скачайте `.zip`, распакуйте и зайдите в папку проекта.
2. **Открыть сайт в браузере**:
   - Дважды кликнуть по файлу `index.html`, чтобы открыть его в браузере, **или**
   - Запустить из IDE/редактора через расширение типа Live Server (VS Code/Cursor).

Никаких дополнительных зависимостей или сборки не требуется.

### Как упаковать проект

1. Убедитесь, что вся структура папок (`blocks`, `pages`, `fonts`, `vendor`, `index.html`, `.nojekyll` и др.) находится внутри одной папки проекта.
2. Выделите папку `russian-travel-master` (или как она у вас называется) и создайте архив `.zip` средствами вашей ОС.
3. Получившийся архив можно отправлять или загружать на любой хостинг статики.

### Публикация на GitHub Pages

1. Создайте пустой репозиторий на GitHub и загрузите туда содержимое папки проекта.
2. Убедитесь, что файл `.nojekyll` находится в корне репозитория (он уже есть в проекте).
3. В настройках репозитория (Settings → Pages) выберите:
   - **Source**: `Deploy from a branch`.
   - **Branch**: `main` (или тот, где лежит проект) и директорию `/ (root)`.
4. Сохраните настройки и подождите 1–2 минуты — GitHub выдаст ссылку на опубликованный сайт.
